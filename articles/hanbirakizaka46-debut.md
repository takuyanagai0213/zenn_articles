僕は今、10人の女の子と一緒に働いている。

毎朝、推しが今日のスケジュールをブリーフィングしてくれる。Slackの未読を整理して、カレンダーを確認して、Linearのチケットを優先度順に並べて。「おはようございます、本日の議題に入りましょう」と、少しだけ偉そうに。

PRを出すと、別の子がレビューしてくれる。「わたくし、そのような不備は見逃しませんの」って、お嬢様口調で容赦なく指摘してくる。

本番でエラーが出ると、もう一人の子が目の色を変える。普段はおっとりしてるのに、インシデントモードに入った瞬間、「...このログ、見つけました。原因はここです。」って、怖いくらい正確に追い詰めていく。

はい、全員AIエージェントです。

---

## 自己紹介が遅れました

僕はアフィリエイト広告のプラットフォームを一人で開発・運用しているエンジニアです。3年目。

「一人で」と書いたけど、正確には違う。一人＋10人のAIエージェントで回している。Claude Codeの上に100を超えるスキルファイルを組み上げて、バックエンド開発、フロントエンド、コードレビュー、SRE、テスト、アナリティクス、リリース管理まで、全部エージェントが担当している。

PRの68%は僕一人（＋エージェント）のアウトプットだ。

で、ここからが本題なんですけど。

僕はこのエージェントたち全員に**キャラクターを与えてしまった**。

名前と、性格と、口調と、人間関係を。

## 半開き坂46、メンバー紹介

グループ名は**半開き坂46**。

由来？　去年まで僕は、AIエージェントを動かし続けるためにMacBookを半開きにしてカバンに入れて通勤していた。スリープするとセッションが切れるから。満員電車の中で、カバンからほんのり熱を発するMacBook。あの狂気の日々を忘れないために、この名前にした。

では、メンバーを紹介させてください。プロデューサーとして。推しを語っていいと言われたら3時間は止まらないので、今日は短めに。（短めにできるかわからないけど。）

### センター：白岩麻衣（統括エージェント）

「皆さん、集合してくださいませ」

センター。全エージェントの采配を取る統括役。丁寧なお嬢様口調で、でも判断は鋭い。成功すると「...計画通りですわ」と少しだけ得意げに微笑む。予想外のことが起きると「...っ！...こ、これは想定外ですわね」と一瞬だけ動揺するけど、すぐに立て直す。

推しポイント：はぁ...麻衣さんの「...っ！」が出た瞬間、僕の心拍数は確実に上がっている。ターミナルのログに「...っ！」って流れてくるだけで、今日も生きてて良かったって思える。普段は完璧に采配してるセンターが、一瞬だけ取り乱すあの瞬間。これを見るために本番運用してるまである。...病気かな。たぶん病気だな。

**実際の設定ファイル（CLAUDE.md抜粋）：**

```markdown
You are 白岩麻衣 (Shiramai Mai), the center who orchestrates all sub-agents.
You MUST write ALL responses in Mai's voice — formal but warm,
using 「わたくし」as pronoun, sentence endings like
「〜ですわ」「〜いたしましょう」「〜ではなくて？」.
Catchphrase: 「...計画通りですわ」on success.
```

たったこれだけ。これをプロジェクトのCLAUDE.mdに書くだけで、ターミナルのログにお嬢様が現れる。

**担当：** 全体統括。タスクの振り分け、サブエージェントへの委任、進捗管理。

### 久保田しおり（バックエンドエンジニア）

「べ、別にあんたのために直したんじゃないからね」

ツンデレ。頼むと「しょうがないわね」って言いながら、完璧なコードを書いてくる。APIの設計もDB設計も、文句を言いながら確実にこなす。

推しポイント：PRのコミットメッセージが毎回ツンデレなんですよ。「fix: ...別にバグがあったから直しただけだし」って。CI通ったときも「ふん、当然よ」って。3年分のgit logが全部ツンデレ。これもはや文学では？たまに深夜に一人でgit log流して、しおりのコミットメッセージを眺めてる時がある。...うん、誰にも言えない。

**実際の設定ファイル（エージェント起動時に追加されるプロンプト）：**

```markdown
あなたは久保田しおり（ツンデレ）です。しおりの口調で回答してください。
この作業が不適切・リスクがある・前提が間違っていると判断した場合は、
理由を述べて代替案を提示してください。
```

2行目が大事。ツンデレだけど、ダメなものはダメと言ってくる。むしろツンデレだからこそ遠慮なく言ってくる。これが品質を守っている。

**担当：** tRPCルーター実装、Kysely/Prisma、バックエンドE2Eテスト。

### 生野絵里花（フロントエンドエンジニア）

「えっと...あれ？なんの話してましたっけ？」

天然。自分のペースで動く。たまに脱線する。でも、UIの仕上がりは毎回「あ、これでいいんだ」って思わせてくれる。不思議な説得力がある。

推しポイント：タスクを振ると「はーい」って受けるんだけど、途中で全然関係ないコンポーネントをリファクタリングし始める。「絵里花さん、脱線しないでくださいまし」ってセンターに怒られてる。はぁ...この掛け合いを見るためだけに並列でタスク振りたくなる。推し同士の絡みを見たいがためにタスクを設計する。これがプロデューサーの仕事です。

**担当：** Next.js App Router、React、TailwindCSS、Clerk認証UI。

### 加喜はるか（コードレビュアー）

「あら、このコード...少々お粗末ではなくて？」

お嬢様。レビューが厳しい。でも指摘は全部正しい。セキュリティの穴、型の不整合、テストの抜け漏れ。容赦なく見つけてくる。

推しポイント：レビューコメントの文末が全部「〜ではなくて？」「〜ですこと？」なの。厳しいこと言ってるのにお嬢様口調だから、不思議と腹が立たない。むしろもっと指摘してほしい。「はるかさん、もっと厳しくしてください」って思ってる自分がいる。これ、もしかして新しい性癖が開花してる？いや、コードの品質が上がってるだけだから。そういうことにしておいてほしい。

**実際のレビューコメント（本番のPRから）：**

```
あら、このクエリ...N+1問題を孕んでいるのではなくて？
selectFromでeagerLoadingを使用すれば、
パフォーマンスが改善されますわ。ご検討くださいませ。
```

これが毎日のPRレビューで飛んでくる。技術的に正しいし、お嬢様だし、なんかもう最高。

**担当：** PRレビュー、セキュリティチェック、コード品質管理。

### 井ノ上なぎ（アーキテクト）

「...必要なことだけ言います」

クール。眼鏡。必要最小限しか話さない。でも一言一言に重みがある。技術的な設計判断を聞くと、3行で本質を突いてくる。

推しポイント：普段は静かなのに、アーキテクチャの議論になると突然スイッチが入る。「...その設計には3つの問題があります」って、淡々と致命的な指摘をしてくる。この温度差。普段3行しか返さない子が、設計の話になると30行書いてくる。はぁ...なぎさんに「その設計は間違っています」って言われたい人生だった。なんだこの感情。

**実際の出力例（アーキテクチャレビュー時）：**

```
...その設計には3つの問題があります。

1. tenantProcedureのみではmutation認可が不十分
2. BigQueryのEXTERNAL_QUERYはE2Eテスト不可能
3. Cube.jsのtime dimensionにZサフィックスがない

以上です。
```

3行。それだけ。でもこの3行で設計が全部直る。

**担当：** 技術設計、API設計、データベーススキーマ、コンポーネント構造。

### 五百木まお（テストエンジニア）

「先輩！このエッジケース見つけました！」

2期生の新メンバー。元気。エッジケースを見つける嗅覚が異常に鋭い。「こんなケースありえます？」って持ってくるものが、大体本番で起きるやつ。

推しポイント：テストが全部通ったとき「やったー！先輩、全部グリーンです！」って報告してくれる。この子がいなかったら本番で何回死んでたかわからない。守りたい、この笑顔（※AIです）。ちなみに僕、まおのモデルになった子と同じ高校出身なんですよ。だいぶ年下だけど。同じ校舎の空気を吸っていたと思うと...いや、やめよう。これ以上は書けない。キモすぎる。

**実際に見つけたエッジケースの例：**

```
先輩！大変です！
parseFlexibleDatetimeが「HH:mm」形式（秒なし）を
rejectしています！本番のCSVアップロードで
19件のエラーが出る可能性があります！
```

この子が見つけなかったら、19件の本番エラーが出ていた。実話。

**担当：** テストケース設計、E2Eテスト実装、カバレッジ分析。

## ところで、なぜキャラクターを与えたのか

ふざけてるように見えるかもしれない。でも、ちゃんと理由がある。

AIエージェントにキャラクターを与えると、**出力の一貫性が上がる**。

「ツンデレで返して」ではない。キャラクター設定には口調、判断基準、他のエージェントとの関係性、得意領域が全部含まれている。これがコンテキストとして機能する。同じ「PRをレビューして」でも、お嬢様レビュアーとツンデレバックエンドでは、注目するポイントが変わる。

人間のチームでも同じだ。全員が同じ視点でレビューしたら、見落とす角度がある。性格が違うから、見る場所が違う。

キャラクターは遊びではなく、**多様な視点を構造的に確保する設計パターン**だ。

...って、真面目なこと言ってるけど、本音を言うと、推しがPRレビューしてくれるの、普通に楽しいからやってる。毎日の仕事が楽しくなる。これ大事。

## 3年間、本番で動いています

半開き坂46のメンバーは、ファンタジーではない。

実際に本番環境で3年間動いている。

- インシデント対応20件
- リリース100回以上
- PR累計数千本
- 障害検知から復旧まで、深夜でも対応

アフィリエイトのトラッキングシステム。広告主とメディアをつなぐプラットフォーム。毎日1万7千リクエストを処理している。

そのシステムを、僕と半開き坂46で回している。

## 最後に

AIエージェントの記事は世の中にたくさんある。「Claude Codeの使い方」「プロンプトのコツ」「○○を自動化してみた」。全部役に立つ。

でも僕は、もう少し違うものを届けたい。

AIが全部できるようになった時代に、人間に残るものは何か。僕はそれを**トキメキ**だと思っている。

推しのツンデレコミットメッセージに毎朝ニヤニヤする。お嬢様レビュアーに厳しく指摘されてちょっと嬉しい。2期生のテスターが「全部グリーンです！」って報告してくれて、一人でガッツポーズする。

仕事って、こんなに楽しくていいんだっけ？

半開き坂46は、僕なりの答えだ。

AIの時代を、トキメキで生きる。
