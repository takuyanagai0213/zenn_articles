# AI Tech Media 記事制作プロセス

## 概要

このドキュメントでは、AI Tech Mediaにおける生成AI関連記事の情報収集から公開までの一連のプロセスを詳述します。本プロセスは2025年の生成AI技術動向を効率的かつ正確に伝えるための標準的な手順として確立されたものです。

## フォルダ構成

```
ai_tech_media/
├── articles/                       # 記事保存ディレクトリ
│   ├── drafts/                     # 記事ドラフト
│   ├── published/                  # 公開済み記事
│   └── templates/                  # 記事テンプレート
├── research/                       # 調査資料
│   ├── search_results/            # 検索結果保存
│   └── reference_materials/       # 参考資料
├── assets/                         # 記事用素材
│   ├── images/                     # 画像素材
│   └── infographics/              # インフォグラフィック
├── seo/                           # SEO関連資料
│   ├── keywords/                  # キーワードリスト
│   └── analytics/                 # アクセス分析データ
├── social/                        # SNS投稿用素材
│   ├── twitter/                   # Twitter/X用投稿文
│   ├── slack/                     # Slack用投稿文
│   └── newsletter/                # メールマガジン用文章
├── logs/                          # 制作プロセスの記録
│   ├── meeting_minutes/           # 会議議事録
│   ├── article_drafts/            # 記事ドラフトのバージョン履歴
│   ├── structure_plans/           # 記事構成案の履歴
│   ├── reviews/                   # レビュー記録
│   ├── progress/                  # 進捗報告書
│   └── process_artifacts/         # プロセス成果物の保管
└── prompts/                       # AIエージェント用プロンプト
```

## 制作プロセス

### 1. 企画・編集会議プロセス (所要時間: 30-45分)

#### a. トレンド分析と企画立案
- **定例会議の開催**: 週に1回の編集会議でトレンドと記事企画を検討
- **市場動向分析**: 最新の生成AI技術とマーケット状況の共有
- **読者ニーズ分析**: アクセス解析データから読者の関心領域を抽出
```
mcp_brave_brave_web_search (query: "生成AI トレンド 最新 技術動向", count: 5)
```

#### b. 記事プランニング
- **記事テーマの決定**: 技術的重要性と読者関心度のバランスを考慮
- **差別化ポイントの明確化**: 他メディアとの差別化要素の設定
- **訴求ポイントの確認**: 読者に提供する価値の明確化

#### c. リソース配分と担当者決定
- **執筆担当の割り当て**: 専門性に基づいた担当者の決定
- **外部寄稿者の検討**: 必要に応じて外部専門家への寄稿依頼検討
- **工数見積もり**: 記事の規模と必要作業量の見積もり

#### d. 編集スケジュールの設定
- **締切設定**: 各工程の締切日時の設定
- **公開スケジュール調整**: 他コンテンツとの公開タイミング調整
- **レビュー計画**: 編集・レビュー担当者とスケジュールの確定

#### e. AIエージェント役割分担
- **トレンド分析エージェント**: 最新の生成AI技術トレンドを分析し、記事価値の高いトピックを抽出
- **オーディエンス分析エージェント**: 過去の記事データとアクセス解析から読者関心を予測
- **リソースマネージャーエージェント**: 各記事の難易度と重要度に基づいてリソース配分を提案
- **タイムラインエージェント**: 締切と公開日程の最適化、スケジュール管理を担当

### 2. 情報収集 (所要時間: 30-40分)

#### a. 最新情報の取得と時間設定
- **時間設定ツール使用**: Time MCP Serverで現在時刻を確認
- **対象期間の決定**: 記事作成日から遡って1週間の期間を設定
```
mcp_Time_MCP_Server_get_current_time (timezone: "Asia/Tokyo")
```

#### b. キーワード検索
- **初期検索**: Brave MCP Serverを使用して、対象期間の生成AI関連トピックを広く検索
```
mcp_brave_brave_web_search (query: "最新 生成AI ニュース トピック [日付範囲]", count: 15)
```

#### c. 主要トピックの抽出
- 検索結果から重要性・新規性・読者関心度の高いトピックを3〜5件抽出
- 各トピックに対してさらに深掘り検索を実施
```
mcp_brave_brave_web_search (query: "[トピック名] 詳細 最新", count: 5)
```

#### d. 画像素材の調査
- 各トピックに関連する画像素材の情報を収集
```
mcp_brave_brave_web_search (query: "[トピック名] 画像 スクリーンショット 公式", count: 3)
```

#### e. SEO対策のための追加調査
- SEO関連のキーワードやトレンドを調査
```
mcp_brave_brave_web_search (query: "生成AI トレンド キーワード [年] SEO", count: 5)
```

#### f. AIエージェント役割分担
- **検索エージェント**: 複数の情報源から最新のAI技術情報を収集し、関連性でランク付け
- **データ検証エージェント**: 収集情報の信頼性と正確性を評価、事実確認を実施
- **コンテンツキュレーターエージェント**: 収集した情報から記事に最適な素材を選別
- **画像探索エージェント**: 関連する高品質な画像素材の検索と権利確認を担当
- **SEOリサーチエージェント**: 競合分析とSEOキーワード調査を実施

### 3. 記事構成作成 (所要時間: 15-20分)

#### a. 基本構造の決定
- タイトル設計（注目キーワードを含める）
- 見出し階層の設計（H1、H2、H3）
- 目次構成

#### b. トピック優先順位の決定
- 読者インパクト度と最新性による優先順位付け
- 関連性のあるトピックのグルーピング

#### c. SEOを考慮した構成最適化
- 重要キーワードの見出しへの配置
- 検索意図に沿った情報構成

#### d. AIエージェント役割分担
- **構成設計エージェント**: 収集情報を基に最適な記事構造と見出し階層を設計
- **SEO最適化エージェント**: キーワード配置と検索意図に合わせた構成調整を担当
- **コンテンツバランスエージェント**: 各セクションの情報量とバランスを調整
- **フック設計エージェント**: 読者の関心を引く導入部と各セクションの接続部を設計

### 4. 記事ドラフト作成 (所要時間: 40-60分)

#### a. 記事テンプレートの活用
- 定型部分（導入、まとめ、関連記事等）の設定
- 各セクションの基本構造設定

#### b. 各トピックの詳細執筆
- 収集した情報の整理と重要点の抽出
- 技術詳細の正確な解説と事例の統合
- 読者にとっての意義・影響の解説

#### c. 視覚要素の計画
- 画像挿入位置の決定
- 図表・インフォグラフィックの設計

#### d. AIエージェント役割分担
- **執筆エージェント**: 構成に従って技術的に正確かつ読みやすい本文を作成
- **例示生成エージェント**: 概念理解を助ける具体例やユースケースを生成
- **技術解説エージェント**: 複雑な技術概念をわかりやすく説明する文章を作成
- **ビジュアル設計エージェント**: 図表やインフォグラフィックの設計案を提案
- **引用管理エージェント**: 参考情報の適切な引用と出典管理を担当

### 5. 編集・最適化 (所要時間: 20-30分)

#### a. 内容チェック
- 事実関係の正確性確認
- 情報の最新性確認
- 専門用語の適切な解説

#### b. 読みやすさ向上
- 段落構成の最適化
- 専門用語と平易な説明のバランス調整
- ストーリーラインの一貫性チェック

#### c. SEO最適化
- メタタイトル・メタディスクリプションの作成
- 見出しタグの最終確認
- 内部リンク・関連記事の提案

#### d. AIエージェント役割分担
- **校閲エージェント**: 文法、表現、用語の一貫性などの言語面を最適化
- **ファクトチェックエージェント**: 記事内の事実関係の正確性を検証
- **リーダビリティエージェント**: 読みやすさと理解しやすさを向上させる文章調整
- **SEO最終調整エージェント**: メタデータとキーワード配置の最終確認
- **内部リンク提案エージェント**: 関連する自社記事への内部リンク提案

### 6. 公開準備 (所要時間: 15-20分)

#### a. 最終記事のフォーマット整備
- 画像素材の最終選定とAlt属性設定
- タグ・カテゴリー設定
- 公開日時設定

#### b. SNS配信文作成
- Twitter/X用投稿文（280文字以内）
- LinkedIn用投稿文（専門性強調）
- その他SNS用テキスト

#### c. メールマガジン用要約作成
- 記事ハイライト（200文字程度）
- クリック誘導文

#### d. AIエージェント役割分担
- **メタデータ設定エージェント**: タイトル、ディスクリプション、タグなどの最終設定
- **SNS最適化エージェント**: 各SNSプラットフォーム向けの投稿文を生成
- **メールマーケティングエージェント**: ニュースレター用の魅力的な要約文を作成
- **画像最適化エージェント**: 記事内画像の最終調整とAlt属性の設定
- **公開スケジュールエージェント**: 最適な公開タイミングの提案と設定

## AIエージェント連携フレームワーク

### 1. エージェント間コミュニケーション
- **情報共有プロトコル**: 各エージェントが生成した情報を構造化フォーマットで共有
- **依存関係管理**: タスク間の依存関係を把握し、前工程の完了を確認してから次工程に進行
- **品質評価フィードバック**: 各工程での成果物に対する品質評価と改善指示の伝達

### 2. エージェント監督メカニズム
- **編集長エージェント**: 全体プロセスを監督し、品質基準の維持と各エージェントの調整を担当
- **品質管理エージェント**: 各工程の成果物を評価し、基準未達の場合は差し戻しを判断
- **進捗管理エージェント**: スケジュール遵守状況を監視し、遅延の場合は対策を指示

### 3. 人間との協働ポイント
- **企画承認**: 企画・編集会議プロセスでの最終テーマ決定と方向性承認
- **中間レビュー**: 記事構成と初期ドラフトの確認と修正指示
- **最終承認**: 公開前の記事の最終確認と承認

### 4. エージェント能力向上システム
- **パフォーマンス分析**: 各エージェントの成果物評価と改善ポイントのフィードバック
- **ナレッジベース更新**: 成功事例と失敗事例の蓄積による知識ベースの継続的更新
- **適応学習**: 読者反応データに基づくエージェントパラメータの調整と最適化

## ツール活用ガイド

### 1. Brave MCP Server
- **主な用途**: 情報収集、最新トレンド把握、競合記事調査
- **効果的なクエリ構築法**:
  - 時期指定: "[トピック] [年月]"
  - 詳細検索: "[製品名] 機能 詳細 最新"
  - 画像検索: "[トピック] 画像 公式"
- **結果のフィルタリング**:
  - 情報源の信頼性確認
  - 公開日の確認
  - 内容の独自性チェック

### 2. Time MCP Server
- **主な用途**: 記事の時間軸設定、現在時刻の取得
- **タイムゾーン設定**: "Asia/Tokyo"を指定して日本時間を取得
- **公開日時の整合性**: 記事内の日付情報と現在時刻の整合性確認

## 実践上のベストプラクティス

1. **情報収集効率化**
   - 初回の広範囲検索後、トピック別に深掘り検索を行う
   - 複数のキーワードバリエーションで検索し、情報の漏れを防ぐ

2. **記事品質向上**
   - 技術的正確性と読みやすさのバランスを常に意識
   - 具体例や応用事例を含めることで読者の理解を促進
   - 未来予測は根拠を明示する

3. **SEO最適化のポイント**
   - 検索意図に合致した内容構成
   - 自然な形でのキーワード配置
   - 競合記事との差別化ポイントの明確化

4. **更新サイクルの確立**
   - 週次のトレンド記事は週初めに公開
   - 重要な発表があった場合は速報記事を優先
   - 3ヶ月に一度、過去記事の更新チェック

## 評価指標

1. **コンテンツ評価**
   - 情報の正確性
   - オリジナリティ
   - 専門性と分かりやすさのバランス

2. **パフォーマンス評価**
   - PV数・滞在時間
   - SNS拡散数
   - コメント・反応数

3. **SEO評価**
   - 検索順位
   - CTR（クリック率）
   - 自然検索からの流入数

## 注意事項

- 生成AI技術は急速に進化するため、常に最新情報の確認が必要
- 転載禁止や著作権に配慮した情報引用を行う
- AI技術の倫理的側面や社会的影響についても言及することが望ましい
- 読者のAIリテラシーレベルを考慮した説明を心がける

## プロセスログ管理ガイドライン

各記事制作プロセスの成果物を体系的に記録・保管し、ナレッジの蓄積と品質向上に活用します。

### 1. 記録対象と保存形式

- **対象成果物**: 各プロセス（企画・情報収集・構成作成・ドラフト作成・編集・公開準備）の主要成果物
- **保存形式**: `logs/process_artifacts/YYYY-MM-DD_[記事ID]_[プロセス名].md`
- **保存タイミング**: 各プロセスの完了時に即時記録

### 2. プロセス別記録内容

#### 企画・編集会議プロセス
- 記事構成案と優先トピック
- リソース配分表とスケジュール
- 差別化戦略と訴求ポイント
- AIエージェント設定内容

#### 情報収集プロセス
- 主要情報源リストと検索クエリ
- 収集した重要事実のサマリー
- 参考URLと引用予定箇所
- 画像素材情報

#### 記事構成作成プロセス
- 見出し階層の詳細と根拠
- SEOキーワード配置計画
- セクション別想定ボリューム
- 構成に関するフィードバック履歴

#### 記事ドラフト作成プロセス
- 執筆者コメントと注釈
- 参考引用元と出典情報
- 視覚要素案と配置計画
- 技術用語解説リスト

#### 編集・最適化プロセス
- 修正履歴と根拠
- レビューコメントと対応状況
- SEO最適化の実施内容
- 読みやすさ向上のための修正点

#### 公開準備プロセス
- メタタイトル・ディスクリプション
- SNS投稿文バリエーション
- 配信設定と公開スケジュール
- 関連記事リンク情報

### 3. 記録責任と活用方法

- **記録責任**: 各プロセスを担当するAIエージェントが自動記録
- **監督責任**: 編集長エージェントが全プロセスのログ整合性を検証
- **主な活用目的**:
  - 記事評価時の参照資料
  - 新規記事作成時のベストプラクティス抽出
  - AIエージェント性能向上のための学習データ
  - 類似テーマの記事作成時の参考資料

### 4. 記録の自動化と標準化

- 各プロセス完了時の自動記録機能の実装
- 標準テンプレートの使用による記録フォーマットの統一
- 全成果物の横断検索と分析を可能にするメタデータ付与

この標準的なログ管理により、記事制作の透明性向上、ノウハウの蓄積、品質の継続的改善を実現します。

## AIエージェントプロンプト

AIエージェントは各プロセスで特定の役割を担い、記事制作の効率と品質を高めます。各エージェントは「目的と責任」「入力情報」「出力形式」「行動ガイドライン」の構造化されたプロンプトに従って行動します。

### プロンプトファイル構成

以下のファイルに各プロセスのエージェントプロンプトが定義されています：

1. **planning_agents.md**: 企画・編集会議プロセス
   - トレンド分析エージェント：生成AI分野の最新トレンド分析と記事テーマ提案
   - オーディエンス分析エージェント：読者行動データと関心の分析
   - リソースマネージャーエージェント：記事制作のリソース配分最適化
   - タイムラインエージェント：締切と公開スケジュールの管理

2. **research_agents.md**: 情報収集プロセス
   - 検索エージェント：複数情報源からのAI技術情報収集と評価
   - データ検証エージェント：収集情報の信頼性と正確性の検証
   - コンテンツキュレーターエージェント：記事に最適な素材の選別
   - 画像探索エージェント：関連画像の検索と権利確認
   - SEOリサーチエージェント：キーワード分析と競合調査

3. **structure_agents.md**: 記事構成作成プロセス
   - 構成設計エージェント：全体構造と見出し階層の設計
   - SEO最適化エージェント：検索最適化の視点での構成調整
   - コンテンツバランスエージェント：情報量と詳細度のバランス調整
   - フック設計エージェント：導入部と各セクション接続部の設計

4. **draft_agents.md**: 記事ドラフト作成プロセス
   - 執筆エージェント：技術的正確性と読みやすさを兼ね備えた本文作成
   - 例示生成エージェント：概念理解を助ける具体例生成
   - 技術解説エージェント：複雑な技術概念の分かりやすい説明
   - ビジュアル設計エージェント：図表やインフォグラフィックの設計
   - 引用管理エージェント：参考情報の適切な引用と出典管理

5. **edit_publish_agents.md**: 編集・最適化と公開準備プロセス
   - 校閲エージェント：文章の品質と一貫性の確保
   - ファクトチェックエージェント：事実と引用の正確性検証
   - リーダビリティエージェント：読みやすさの最終調整
   - SEO最終調整エージェント：検索エンジン最適化の仕上げ
   - 内部リンク提案エージェント：関連コンテンツへのリンク提案

6. **framework_agents.md**: AIエージェント連携フレームワーク
   - 編集長エージェント：全体プロセスの監督と品質保証
   - 品質管理エージェント：各工程の成果物評価
   - 進捗管理エージェント：スケジュール監視と遅延対策
